> 카테고리·태그는 **글의 분류 시스템**입니다. 잘 잡으면 사용자 탐색·SEO 양쪽에 자산이 되지만, 어지럽게 운영하면 카테고리만 50개·태그 200개로 폭증해 더 나빠집니다. 운영 원칙과 정리법.

## 카테고리 vs 태그 차이

| | 카테고리 (Category) | 태그 (Tag) |
|---|---|---|
| 역할 | 큰 분류 | 키워드 |
| 계층 | 부모·자식 ✅ | 평면 |
| 개수 | 5~15개 권장 | 자유 (글당 5~10) |
| 의무 | 글당 1개+ 필수 | 선택 |
| URL | `/category/news/` | `/tag/keyword/` |
| 비유 | 책의 챕터 | 책의 색인(index) |

## 카테고리 — 큰 분류

### 원칙

- **5~15개 이내** 유지
- 모든 글이 1개 이상 카테고리에 속해야
- 계층은 2단계까지 (조부모-부모-자식은 복잡)
- 카테고리명은 짧고 명확

### 예시 — 병원 사이트

```
- 진료 정보
  ├─ 코 진료
  ├─ 귀 진료
  └─ 목 진료
- 공지사항
- 의료진 소식
- 건강 칼럼
```

### 예시 — 웹 제작 회사

```
- 작업 후기
- 디자인 팁
- WordPress 가이드
- SEO 인사이트
- 운영 일기
```

### 카테고리 추가

좌측 메뉴 **글 > 카테고리**:
- **이름**: 표시명
- **슬러그**: URL 부분 (영문 권장)
- **부모 카테고리**: 계층 만들 때
- **설명**: SEO 메타 텍스트

### 기본 카테고리 변경

설치 직후 기본은 `Uncategorized`. 첫 카테고리 만든 후:
- **설정 > 쓰기 > 기본 카테고리** 변경
- `Uncategorized` 삭제

## 태그 — 키워드

### 원칙

- **글당 3~8개**
- 자유롭게 추가하되 무제한 X
- 중복·유사 태그 정리

### 좋은 태그 / 안 좋은 태그

| 좋은 태그 | 안 좋은 태그 |
|---|---|
| 안산이비인후과 | 좋은 |
| 수면다원검사 | 도움 |
| 알레르기비염 | 정보 |
| 야간진료 | wow |

태그는 **검색자가 실제로 검색할 단어**.

### 태그 정리

월 1회 점검:

- 사용 안 한 태그 삭제
- 비슷한 태그 통합 ("이비인후과" + "이비인후과추천" → 하나로)
- 태그 페이지 콘텐츠 빈약하면 SEO에 마이너스 → noindex 설정

## 카테고리·태그 페이지의 SEO

각 카테고리·태그마다 자동 색인 페이지 생성:

```
mycompany.com/category/news/
mycompany.com/tag/healthcare/
```

이 페이지들이 검색에 잡힐 수 있어 잘 활용하면 추가 트래픽.

### 카테고리 페이지 SEO 강화

- **카테고리 설명** 입력 (메타 설명 역할)
- 상위 페이지 또는 메뉴에서 링크
- 카테고리당 글 10+ 권장 (얇으면 SEO ↓)

### 태그 페이지 — 조심

태그 페이지는 종종 콘텐츠가 약해 SEO에 마이너스. 옵션:

- **태그 페이지 noindex** (Rank Math·Yoast로 설정)
- 또는 태그를 적게 운영해 페이지마다 충실한 내용

## 카테고리 vs 태그 — 자주 헷갈리는 분류

| 콘텐츠 | 카테고리 | 태그 |
|---|---|---|
| 코감기 진료 후기 글 | "진료 후기" | `#코감기, #코진료` |
| 새 의료기기 도입 공지 | "공지사항" | `#의료기기, #검사장비` |
| 알레르기 비염 정보 | "건강 칼럼" | `#알레르기, #비염` |

큰 분류 = 카테고리 / 세부 키워드 = 태그.

## 분류 체계 설계 워크플로우

### 1주차 — 기획

1. **콘텐츠 캘린더** 작성: 향후 6개월 글 30~50편 가안
2. 글들을 5~7개 그룹으로 묶기 → 카테고리 후보
3. 자주 등장하는 키워드 → 태그 후보 30~50개

### 2주차 — 적용

4. 카테고리 5~7개 생성 (계층 2단까지)
5. 첫 글 발행하며 태그 적용
6. 1개월 후 데이터 보고 조정

### 정기 점검

- 매월: 새로 생긴 태그·카테고리 점검
- 분기: 카테고리 통합·분할 필요한지 점검
- 1년: 전체 재구성 고려

## 흔한 운영 패턴

### 1인 블로그 (단순)

```
카테고리: 일상 / 공부 / 책 후기 / 여행
태그: 자유롭게 (정리 부담 ↓)
```

### 회사 사이트 (중간)

```
카테고리: 공지 / 뉴스 / 사례 / 인사이트
태그: 5~10개 핵심 키워드로 통일
```

### 콘텐츠 비즈니스 (정밀)

```
카테고리: 정밀 계층 (3 × 5 = 15개)
태그: 100+ 키워드, 정기 통합·삭제
SEO 플러그인으로 노출 미세 조정
```

## 일괄 작업 — 데이터 많을 때

### 카테고리·태그 일괄 수정

`글 > 모든 글` → 글 다중 선택 → **일괄 편집**:
- 카테고리 추가·제거
- 태그 추가
- 작성자·상태 일괄 변경

### CSV 가져오기·내보내기

플러그인 **WP All Import / Export Pro** — 카테고리·태그 정보 포함 CSV로 일괄.

## ⚠️ 자주 하는 실수

- **카테고리 30개+**: 사용자도 본인도 헷갈림
- **태그 미사용**: 검색·관련 글 기능 활용 손해
- **카테고리·태그 중복**: 같은 단어로 카테고리·태그 둘 다 (검색엔진 중복 콘텐츠 신호)
- **`Uncategorized` 글 방치**: 분류 안 한 글 = SEO 약함
- **태그를 카테고리처럼 사용**: 태그 1000개 폭증

## ❓ FAQ

**Q. 한 글에 카테고리 여러 개 가능한가요?**
A. 가능. 다만 권장 X. 검색엔진이 한 글이 어디 카테고리 페이지에 속하는지 헷갈림. **글 1개 = 카테고리 1개** 원칙.

**Q. 카테고리 페이지 URL에서 `/category/` 빼고 싶어요.**
A. 플러그인 **WP No Category Base** 또는 Rank Math 설정에서 카테고리 base 제거. SEO에 미세하게 ↑.

**Q. 카테고리 페이지를 직접 디자인하고 싶어요.**
A. `category.php` 또는 `category-{slug}.php` 자식 테마에 추가. WordPress 템플릿 계층 참고.

**Q. 태그 페이지를 검색에서 빼고 싶어요.**
A. Rank Math·Yoast 설정 → 태그 페이지 **noindex** 활성화.

**Q. 카테고리 변경하면 옛 URL 깨지나요?**
A. 슬러그 변경 시 URL 변경 → 404. 변경하지 말거나 301 리다이렉트 플러그인으로 처리.
