> 디자인이 아무리 빨라도 **들어갈 콘텐츠 자료가 없으면 제작이 멈춥니다.** 로고 원본, 사진, 텍스트가 가장 흔한 지연 원인이에요. 이 체크리스트를 의뢰 초반에 공유하면 제작 기간이 눈에 띄게 짧아집니다.

## 왜 자료 준비가 병목인가

제작은 디자인 → 콘텐츠 채우기 순서가 아니라, **콘텐츠가 있어야 디자인이 완성**됩니다. 실제 문구·사진이 없으면:
- 빈 박스에 의미 없는 더미 텍스트로 시안 → 나중에 안 맞음
- 사진 자리만 비워두고 → 발행 직전 급하게 아무 사진
- 로고 저해상도 → 흐릿하게 박힘

> 💡 자료를 **초반에 한 번에** 받으면, 디자인과 콘텐츠가 처음부터 맞물려 수정이 줄어듭니다.

## 1. 로고 — 가장 자주 막히는 항목

| 필요한 것 | 이유 |
|---|---|
| **벡터 파일** (AI·SVG·EPS) | 어떤 크기로도 깨지지 않음. 가장 이상적 |
| **투명 배경 PNG** (고해상도) | 벡터가 없을 때 차선. 최소 폭 1000px↑ |
| **가로형 / 세로형 / 심볼** 버전 | 헤더·푸터·모바일·파비콘에 각각 필요 |
| **로고 사용 규칙** (있다면) | 여백·색상·금지 사용 등 |

> ⚠️ JPG 로고는 배경이 흰색으로 박혀 있어 거의 못 씁니다. **투명 배경(PNG/SVG)** 으로 받으세요.

## 2. 사진·이미지

| 항목 | 기준 |
|---|---|
| **해상도** | 가로 최소 1500px↑ (Hero용은 2000px↑) |
| **저작권** | 직접 촬영 또는 상업용 라이선스 확보분만 |
| **종류** | 외관·내부·대표·인물·제품/서비스 |
| **형식** | JPG(사진) / PNG(투명 필요 시) |

- 휴대폰 사진도 요즘은 충분히 고해상도지만, **밝고 선명한 것**으로.
- 인물·매장 사진은 **신뢰 신호**라 전환에 직접 도움이 됩니다.
- 사진이 부족하면 유료 스톡(예산 범위 내) 또는 촬영 일정을 미리 잡으세요.

## 3. 텍스트(원고)

각 페이지에 들어갈 글을 **초안이라도** 미리 받습니다. 완벽하지 않아도 됩니다 — 다듬는 건 제작 과정에서 합니다.

- 회사·브랜드 **소개** (한 문단 + 상세)
- **서비스/진료/제품** 설명 (항목별)
- **대표 메시지** (홈 첫 화면에 넣을 한 줄)
- **연혁·자격·인증** (신뢰 요소)
- **자주 묻는 질문** 후보

> 텍스트가 막막하면 [콘텐츠 우선 디자인](/guides/design-content-strategy/)과 [웹사이트 카피라이팅](/guides/design-copywriting/)을 참고해 핵심부터 정리하세요.

## 4. 기본 정보 (NAP 등)

검색 노출과 직결되는 정보라 **정확히** 받아야 합니다.

- 정확한 **상호명** (띄어쓰기까지 통일)
- **주소** (지번/도로명)
- **전화번호** (대표번호)
- **영업시간 / 휴무일**
- **이메일 / 카카오 채널 / SNS** 링크
- (병원·지역업) 네이버 플레이스·지도 링크

> 이 정보는 모든 채널에서 똑같아야 합니다. ([로컬 SEO](/guides/seo-local/)의 NAP 일관성 참고)

## 5. 계정·접근 권한

발행·연동에 필요한 것들입니다. 보안상 **작업 직전에** 안전한 방법으로 전달받으세요.

- **도메인** 관리 계정 (또는 네임서버 변경 권한)
- **호스팅** 계정 (해당 시)
- 기존 사이트가 있다면 **관리자 접근**
- 연동할 **분석/광고 계정** (GA4·네이버·메타 등)

> ⚠️ 비밀번호는 메신저 평문보다 안전한 방법으로. 가능하면 **권한 초대**(관리자 추가) 방식을 쓰세요.

## 6. 참고·취향

방향을 빠르게 맞추는 데 큰 도움이 됩니다.

- **마음에 드는 사이트** 2~3개 + 어느 점이 좋은지
- **피하고 싶은 스타일**
- 선호 **색·톤** (있다면)
- 경쟁사/동종업계 사이트

→ [참고 사이트 찾는 법](/guides/design-reference-sites/) 참고.

## 의뢰자용 — 한 장 체크리스트

- [ ] 로고 (벡터 또는 고해상도 투명 PNG)
- [ ] 사진 (고해상도, 저작권 확보)
- [ ] 페이지별 텍스트 초안
- [ ] 상호·주소·전화·영업시간 (정확히)
- [ ] 이메일·SNS·채널 링크
- [ ] 도메인/호스팅/분석 계정 정보
- [ ] 참고 사이트 2~3개

## ⚠️ 자주 하는 실수

- **로고를 JPG/저해상도로 전달**: 흐릿하거나 흰 배경이 박힘
- **사진을 발행 직전에 줌**: 디자인을 다시 맞춰야 함
- **텍스트를 "알아서 써주세요"**: 업계 지식은 의뢰자가 가장 정확. 초안은 직접
- **상호·주소 표기 제각각**: 검색엔진 혼동 → 로컬 SEO 손해
- **비밀번호 평문 전송**: 보안 위험. 권한 초대 권장

## ❓ FAQ

**Q. 텍스트를 꼭 의뢰자가 써야 하나요?**
A. 초안은 의뢰자가 가장 정확합니다(업계 지식). 다만 다듬기·교정·SEO 반영은 제작 과정에서 합니다. AI로 초안을 잡아도 되지만, 실제 경험·사실은 본인이 채워야 해요.

**Q. 사진이 거의 없어요.**
A. 촬영 일정을 잡거나 상업용 스톡 이미지를 활용합니다. 다만 인물·매장 실사진이 신뢰에 훨씬 효과적이라, 핵심 사진은 직접 찍는 걸 권합니다.

**Q. 로고 벡터 파일이 없는데요?**
A. 제작했던 디자이너에게 원본(AI·SVG)을 요청하세요. 정 없으면 고해상도 투명 PNG로 차선책을 쓰되, 필요 시 로고를 다시 벡터화하는 작업이 들 수 있습니다.

**Q. 자료를 다 모은 뒤에 시작해야 하나요?**
A. 전부는 아니어도 **핵심(로고·대표 텍스트·기본 정보)** 은 시작 전에 있어야 합니다. 나머지는 제작과 병행해도 되지만, 늦을수록 지연됩니다.
