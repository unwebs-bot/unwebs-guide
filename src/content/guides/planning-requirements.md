> 사이트 제작 의뢰에서 가장 큰 시간 낭비는 **의뢰 시작 후 요구사항이 계속 바뀌는 것**입니다. 미리 12가지만 정리하면 작업 중 수정 횟수가 절반으로 줄어요. 의뢰자·제작자 양쪽이 함께 채울 체크리스트.

## 왜 사전 정리가 중요한가

### 사전 정리 X

```
의뢰 시작 → 디자인 → "이 색 다른 걸로" 
         → "메뉴 추가해주세요"
         → "기능 하나 더"
         → 일정 2배·비용 1.5배 + 양쪽 피로
```

### 사전 정리 O

```
요구사항 합의 → 디자인 → 검수 (이미 합의된 범위)
            → 미세 조정만
            → 일정·비용 예측 가능
```

30분 사전 미팅 = 작업 10시간 절약.

## 12가지 합의 항목

### 1. 목적

**한 줄로**:
- "검색해서 찾아온 고객이 예약하게"
- "회사 신뢰도를 높여 의뢰 받기"
- "포트폴리오 보여주고 영업"
- "온라인 판매"

목적이 명확해야 모든 결정의 기준이 됨.

### 2. 타겟 사용자

- **누구**: 20·30대 직장인 / 50·60대 시니어 / 기업 담당자
- **언제**: 모바일 출근길 / PC 회사 점심시간 / 저녁 가족
- **무엇을 원하나**: 정보 / 비교 / 즉시 구매 / 예약

같은 "병원 사이트"라도 타겟에 따라 디자인 완전 다름.

### 3. 핵심 행동 (Conversion)

방문자가 사이트에서 **최종 하길 원하는 행동**:

- 예약·문의 폼 제출
- 전화 클릭
- 상품 구매
- 자료 다운로드
- 회원 가입

**1개 또는 2개**가 정답. 5개+이면 우선순위 ↓.

### 4. 페이지 목록

[사이트맵 가이드](/guides/planning-sitemap/) 참고.

미리 합의:
- 핵심 페이지 5~10개
- 향후 확장 페이지

### 5. 기능 목록

- 회원가입·로그인?
- 결제?
- 예약 시스템?
- 멤버십 차등?
- 다국어?
- 검색?

각 기능 = 시간·비용 ↑. 진짜 필요한 것만.

### 6. 콘텐츠 준비

가장 자주 일정이 밀리는 영역.

#### 사전 준비 (의뢰자)

- [ ] 회사 로고 (벡터·SVG)
- [ ] 제품·서비스 사진 (고해상도)
- [ ] 회사 소개 텍스트
- [ ] 서비스 설명 텍스트
- [ ] 직원·팀 사진 + 약력
- [ ] 후기·사례
- [ ] 약관·개인정보 처리방침 초안

부족하면 **콘텐츠 작성·촬영도 별도 비용**.

### 7. 디자인 방향

#### 참고 사이트 3~5곳

- 좋은 점: 무엇이 좋은지 한 줄
- 따라하지 말 것: 피하고 싶은 것

#### 톤 키워드

- 전문적 / 친근 / 모던 / 고급 / 활기찬
- 명확한 한 단어 (혼합 X)

#### 색·폰트 가이드

- 회사 브랜드 컬러 (HEX)
- 선호 폰트 또는 시스템 기본

### 8. 예산

#### 일반 기준

| 사이트 유형 | 예산 |
|---|---|
| 단일 페이지 랜딩 | ₩100~300만 |
| 5페이지 회사 사이트 | ₩200~500만 |
| 10페이지 비즈니스 | ₩500~1,000만 |
| 쇼핑몰 (기본) | ₩500~2,000만 |
| 복잡한 시스템 | 별도 |

**유지보수 비용**(연간 10~20%) 별도 합의.

### 9. 일정

#### 표준

- 단순 랜딩: 2~4주
- 일반 비즈니스: 4~8주
- 복잡한 시스템: 2~6개월

#### 변동 요소

- 콘텐츠 준비 지연 = 일정 지연
- 디자인 검수 시간 = 일정 일부
- 의뢰자 반응 속도 = 일정 결정

### 10. 도구·플랫폼

미리 합의:
- WordPress vs 아임웹 vs 정적
- 호스팅 선택
- 도메인 (이미 있는지·새로 살지)

[WordPress 처음 시작하기](/guides/wp-getting-started/) 참고.

### 11. SEO·마케팅 요구

- 검색엔진 등록까지?
- 광고 운영도 포함?
- 콘텐츠 작성 지원?
- 정기 운영 지원?

기본 SEO 셋업 외엔 별도 비용.

### 12. 인수인계·유지보수

- 완성 후 사용법 교육 (몇 시간?)
- 매뉴얼 작성
- 사후 수정 횟수·범위
- 정기 유지보수 계약 여부

```
5. 기능: ___________
6. 콘텐츠 (의뢰자 준비 vs 제작자 작성): ___________
7. 디자인 방향: 참고 사이트·톤·색
8. 예산: ___________
9. 일정: ___________

```

PDF로 정리 → 양쪽 사인 → 작업 시작.

## 흔한 분쟁 — 사전 방지

### "이 정도는 무료 아닌가요"

명시:
- 수정 횟수 (예: 디자인 시안 3회, 본 작업 3회)
- 수정 범위 (디자인 색상 변경 OK, 페이지 추가 추가 비용)
- 무료 vs 추가 비용 경계

### "급한 변경 부탁해요"

명시:
- 응답 시간 (영업일 24시간 이내 등)
- 긴급 작업 추가 비용

### "유지보수도 평생 무료"

명시:
- 완성 후 30일 무료 버그 수정
- 그 이후 시간당 또는 월 정액

## 의뢰자에게 — 좋은 의뢰

훌륭한 제작자를 만나려면:

- **목적 명확**: "예쁘게"가 아니라 "예약 100건/월"
- **예산 솔직**: 100만원·1000만원 차이 큼
- **참고 사이트**: 말로 설명보다 보여주기
- **빠른 응답**: 늦으면 본인 일정도 늦음
- **신뢰**: 매일 간섭은 결과·관계 양쪽 ↓

## 제작자에게 — 좋은 제작

훌륭한 결과를 만들려면:

- **수동적 받기 X**: 의뢰자가 모르는 걸 안내
- **솔직한 견적**: 가능·불가능·추가 비용 명시
- **체크포인트**: 30%·50%·80% 시점에 검수
- **문서화**: 결정사항 기록·공유
- **사후 지원**: 첫 30일은 책임

## ⚠️ 자주 하는 실수

- **구두 합의만**: 분쟁 시 증거 X
- **모든 디테일 사전에**: 일부는 작업 중 발견. 합의서는 큰 틀만
- **너무 욕심**: 100% 만족 X. 80% 좋으면 출시 후 보강
- **예산 협상 잘못**: 너무 싸면 품질 ↓, 너무 비싸도 가성비 ↓
- **유지보수 무시**: 사이트는 한 번에 끝나는 게 아님

## ❓ FAQ

**Q. 의뢰 전 합의서 작성을 누가 주도?**
A. 의뢰자가 양식 채우고, 제작자가 검토·보완이 표준. 처음부터 양쪽 함께라면 더 좋음.

**Q. 견적은 어떻게 받아야 합리적?**
A. 3~5곳 견적 비교. 최저가 X, 평균 가성비. 포트폴리오·후기 함께 검토.

**Q. 작업 중 수정 요청이 무한히 가능한가요?**
A. 합의서에 명시한 만큼만. 그 이상은 추가 비용.

**Q. 콘텐츠를 제작자가 작성해줄 수 있나요?**
A. 가능하지만 별도 비용. 직접 작성이 비용 ↓·정확성 ↑.

**Q. 완성 후 사이트 운영은 누가?**
A. 의뢰자 직접 (인수인계 받음) 또는 정기 유지보수 계약.
