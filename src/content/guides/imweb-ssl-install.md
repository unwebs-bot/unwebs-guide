> `https://`로 시작하는 사이트는 **SSL 인증서**가 적용된 사이트입니다. 회사·병원 홈페이지에서 SSL은 사실상 필수예요 — 검색엔진 노출, 사용자 신뢰, 법적 의무 모두에 영향. 이 글에서는 SSL 종류·비용·설치 절차를 한 번에 정리합니다.

## 시작하기 전에

- **개인 도메인 연결 완료** 필수. [도메인 연결 가이드](#) 먼저.
- **요금제 확인**: SSL은 Starter 이상부터 가능.
- **소요 시간**: 신청 5분 + 발급 대기 최대 24시간.

---

## SSL이란?

브라우저와 서버 사이의 통신을 **암호화**하는 기술. 인증서가 적용되면:
- 주소창에 🔒 자물쇠 아이콘
- URL이 `http://`에서 `https://`로
- 통신 내용이 해커에게 노출돼도 해독 불가

### 적용 의무 (법적)

다음 중 하나라도 해당되면 **법령상 SSL 의무**:
- 회원가입·로그인 기능
- 결제 처리
- 개인정보 수집(이름·연락처·이메일 포함 폼)
- 게시판 글쓰기

> 즉, 거의 모든 비즈니스 사이트가 의무. 안 깔면 과태료 대상.

### 설치 안 했을 때 손해

- 검색엔진(특히 네이버) 색인 우선순위 ↓
- 크롬 브라우저 "안전하지 않은 사이트" 경고 표시
- 브라우저 위치정보·결제 API 일부 차단
- 사용자 신뢰도 ↓ (방문 즉시 이탈)

### 아임웹 기본 도메인은?

`mysite.imweb.me`처럼 아임웹 기본 도메인은 **기본 SSL 무료 제공**. 개인 도메인(`mycompany.com`) 사용 시에만 별도 신청 필요.

## SSL 종류 비교

| 유형 | 요금 | 지원 도메인 | 추천 대상 |
|---|---|---|---|
| **Basic SSL** | 35,000원/년 | 1개 (단일 도메인) | 일반 회사·병원·소개형 사이트 (대부분 이걸로 충분) |
| **Global SSL** | 170,000원/년 | 1개 | 금융·법무 등 **기관 정보 공식 표시**가 필요한 사이트 |
| **Wildcard SSL** | 220,000원/년 | 무제한 2차 도메인 | `*.mycompany.com` 형태의 다수 서브도메인 운영 시 |

### 어떤 걸 골라야 할까?

- **대부분의 비즈니스 사이트** → **Basic SSL**. 가성비·검색 영향 동일.
- **금융·법무·정부 협력 기관** → Global SSL. 주소창에 기관명 표시되어 신뢰도 ↑.
- **여러 서브도메인** (`shop.`, `admin.`, `blog.`) → Wildcard SSL.

> 💡 사이트가 1개고 서브도메인 안 쓴다면 그냥 **Basic** 가세요. Global·Wildcard는 오버스펙.

## SSL 신청 절차

### 1단계: 도메인 연결 확인

신청 메뉴는 **개인 도메인이 연결된 후에만** 활성화됩니다. 미연결 시 [도메인 연결 가이드](#) 먼저.

### 2단계: SSL 신청 메뉴 진입

1. 관리자 → **환경설정 > 도메인·SSL > 보안 서버(SSL)**.
2. **관리하기** 버튼.
3. 신청할 도메인 선택.
4. **SSL 신청** 버튼.

<figure class="img-placeholder">📷 캡처 예정: SSL 신청 화면</figure>

### 3단계: 종류 선택·결제

1. Basic / Global / Wildcard 중 선택.
2. 결제 카드 등록 (없으면 새로 등록).
3. 결제.

### 4단계: 발급 대기

- **Basic**: 약 5~10분
- **Global**: 1~3일 (기관 확인 필요)
- **Wildcard**: 약 1시간

발급 완료 시 등록 이메일로 알림. 사이트 주소 앞에 `https://` 붙여 접속해 자물쇠 아이콘 확인.

## SSL 적용 후 점검

### 1. 자물쇠 아이콘 확인
브라우저 주소창에 🔒 표시되는지.

### 2. `http://`로 접속 시 자동 리다이렉트
`http://mycompany.com` 입력 → `https://mycompany.com`으로 자동 이동되는지.

### 3. 콘텐츠 깨짐 확인
다음 항목이 정상 작동하는지:
- 모든 페이지 정상 로드
- 이미지 표시 (Mixed Content 경고 없음)
- 결제·로그인 기능
- 외부 위젯 (지도·SNS 임베드)

> ⚠️ 코드 위젯에서 `http://`로 시작하는 외부 스크립트·이미지를 불러왔다면 **Mixed Content 오류**로 차단됩니다. **`https://`로 변경** 필수.

## 자주 발생하는 문제

### 1. SSL 적용 후 사이트 접속 안 됨
- 도메인 네임서버가 잘못 설정됐을 가능성. **할당된 네임서버 1~4차** 다시 확인.
- 도메인 호스팅사에서 입력한 네임서버를 캡처해 아임웹 고객센터에 문의.

### 2. SSL 신청했는데 발급 안 됨
- 도메인 소유 정보가 비공개로 설정되어 있으면 차단됩니다. 도메인 호스팅사에서 WHOIS 공개로 변경.
- 도메인이 만료 직전이면 발급 거부. 도메인 갱신 후 재신청.

### 3. SSL 적용 후 이미지·스크립트 안 보임
- Mixed Content 문제. 코드 위젯·외부 스크립트의 `http://`를 모두 `https://`로 변경.
- 변경할 수 없는 외부 리소스는 다른 서비스로 교체.

### 4. 타사에서 산 SSL을 아임웹에 쓸 수 있나?
- **불가능**. 아임웹은 자체 SSL만 적용 가능. 다른 호스팅에서 산 SSL은 사용 불가.

---

## ⚠️ 자주 막히는 포인트

- **개인 도메인 연결 전에 SSL 신청 시도** → 메뉴가 활성화 안 됨. 도메인 연결 먼저.
- **`http://`로 적힌 외부 링크 그대로 둠** → Mixed Content 오류로 이미지 안 보임. 모두 `https://` 변경.
- **Wildcard 필요 없는데 비싼 거 결제** → Basic SSL이면 충분.
- **만료 안내 무시** → 만료 시 사이트가 "안전하지 않음" 경고. **자동 갱신 설정** 권장.

## ❓ FAQ

**Q. SSL을 안 사고도 https 만들 수 있나요?**
A. 아임웹은 무료 SSL을 제공하지 않습니다(Let's Encrypt 같은 무료 인증서 미지원). 다른 솔루션(WordPress 등)에서는 무료 SSL이 가능하지만, 아임웹은 유료 SSL만.

**Q. SSL 만료 안내가 왔는데 자동 갱신이 안 됐어요.**
A. 결제카드 정보가 만료됐을 가능성. **환경설정 > 결제** 에서 카드 정보 확인. 갱신 안 되면 사이트가 한순간에 "안전하지 않음"으로 전환되니 주의.

**Q. 사이트 만들고 며칠 운영했는데 그동안 SSL 없었어요. 검색에 영향이?**
A. 색인 우선순위가 일시적으로 낮았을 수 있지만 SSL 적용 후 정상 회복됩니다. Search Console에서 색인 요청.

**Q. SSL 적용 후 결제·로그인이 더 잘 되나요?**
A. SSL은 통신 보안이므로 결제·로그인 자체 속도엔 영향 없습니다. 다만 SSL 미적용 사이트는 일부 브라우저가 결제·로그인 기능을 차단하므로 결과적으로 "더 잘 됨".

**Q. 도메인 1개에 여러 SSL 신청 가능?**
A. 동일 도메인에는 1개만. 종류 변경(Basic → Global)을 원하면 기존 SSL 해지 후 새로 신청.
