> 아임웹은 GA 없이도 자체 통계를 제공합니다. 매출·방문자·유입 경로·기기 비율을 **관리자 페이지 안에서 즉시** 확인 가능. 다만 GA만큼 정교하지 않아서 **GA·메타 픽셀과 병행** 사용이 정답.

## 시작하기 전에

- **요금제 확인**: 일부 고급 통계는 Pro 이상 요금제.
- **데이터 반영 시간**: 매출은 4시간, 방문자 통계는 약 1시간.
- **GA·메타 픽셀과 동시 사용 권장**: 각각 강점이 다름.

---

## 어디서 보나?

관리자 → **사용자 관리 > 통계** (또는 좌측 메뉴의 통계 그룹).

## 주요 메뉴 4개

### 1. 매출 (쇼핑몰·결제 사이트)

- **기간별**: 일/주/월 매출, 환불, 순매출
- **상품별**: 상품 단위 판매 개수·매출 추이
- **환경별**: 결제 수단 비율, PC/모바일 비율

**데이터 갱신**: 4시간 주기.

> 💡 **주의**: 쇼핑몰 운영이 아니면 매출 메뉴는 비어있어요. 일반 비즈니스 사이트는 다음 항목들 활용.

### 2. 방문자

- **방문자 수**: 일·주·월별
- **신규 방문 vs 재방문 비율**
- **체류 시간**
- **페이지뷰 수**

### 3. 유입 경로

- **직접 방문**: URL 직접 입력 또는 즐겨찾기
- **검색**: 네이버·구글·다음 등 검색엔진
- **외부 사이트**: 다른 사이트의 링크에서 유입
- **SNS·메신저**: 인스타그램·카카오톡 등

> ⚠️ **"직접 방문" 비중이 높다면** UTM 태그 미사용 가능성. 광고 클릭이 직접 방문으로 잡힐 수 있어요.

### 4. 페이지별 통계

- 메뉴별 방문자 수·체류 시간
- 가장 인기 페이지 / 가장 짧게 머무는 페이지
- 이탈률 (들어왔다가 바로 나간 비율)

## 매출 vs 기간별 분석 — 다른 점 주의

매출 메뉴와 기간별 분석은 **집계 기준이 달라 값이 다를 수 있어요**.

| 항목 | 매출 메뉴 | 기간별 분석 |
|---|---|---|
| 환불 처리 | 포함 | 제외 (순매출만) |
| 쿠폰 사용 | 포함 | 환불 미고려 |
| 부분 취소 | 정확히 차감 | 정확도 낮음 |
| 갱신 주기 | 4시간 | 약 1시간 |

> 💡 **공식 보고용은 "매출" 메뉴**, 일별 추적은 "기간별 분석" 활용.

## 자주 보는 지표 5개

### 1. 일일 방문자 (DAU)
사이트가 살아 있는지 가장 빠르게 확인.

### 2. 주간 신규 방문자
신규 유입 확보가 잘 되고 있는지.

### 3. 유입 경로 비율
어디서 사람들이 오는지 → 광고·SEO 예산 배분 근거.

### 4. 모바일 vs PC 비율
보통 모바일이 60~80%. 모바일 최적화가 우선임을 재확인.

### 5. 이탈률 높은 페이지
방문자가 들어왔다가 바로 나가는 페이지 → 우선 개선 대상.

## 자체 통계 vs Google Analytics

| 항목 | 아임웹 자체 통계 | Google Analytics (GA4) |
|---|---|---|
| **설치** | 자동 | 별도 설치 필요 |
| **갱신 주기** | 1~4시간 | 실시간 ~ 24시간 |
| **상세도** | 기본적 | 매우 상세 (이벤트·전환·코호트) |
| **데이터 보관** | 6개월 ~ 1년 (요금제별) | 최대 14개월 (무료) |
| **커스텀 분석** | 제한적 | 무제한 |
| **비용** | 요금제 포함 | 완전 무료 |

> 💡 **결론**: 자체 통계는 빠른 확인용, GA는 깊은 분석용. **둘 다 켜두세요**.

## 통계로 의사결정하기 — 실전 예시

### 예시 1: "광고 효율이 안 좋아요"

체크 순서:
1. 유입 경로에서 광고 매체별 비율 확인
2. 페이지별 통계에서 광고 랜딩 페이지의 이탈률
3. UTM 태그 분석 (GA에서)

→ **이탈률 높으면 랜딩 페이지 개선, 낮으면 광고 카피·타겟 조정**.

### 예시 2: "사이트는 만들었는데 사람이 없어요"

체크 순서:
1. SEO 등록 완료 여부 ([SEO 가이드](#) 참고)
2. 검색 유입 비율 (낮으면 SEO 부족)
3. 외부 사이트 유입 (블로그·SNS 연동 활성화)

### 예시 3: "방문자는 많은데 매출이 없어요"

체크 순서:
1. 페이지별 통계: 어디까지 보고 이탈하는지
2. 결제 페이지의 이탈률
3. 모바일 사용자의 결제 완료율 (모바일 결제 UX 문제 가능성)

## ⚠️ 자주 막히는 포인트

- **"직접 방문"이 너무 많음** → UTM 태그 안 붙은 광고·외부 링크 때문. [UTM 가이드](#) 참고.
- **본인 IP 카운트** → 본인이 자주 들어가면 데이터 오염. **시크릿창** 사용.
- **모바일 통계가 부정확** → 일부 모바일 브라우저(인앱 브라우저)는 추적 차단. 정상.
- **매출이 GA와 다름** → 정상. 아임웹은 결제 시점 기준, GA는 이벤트 기준. 매출은 아임웹 데이터를 정공식으로.
- **통계 메뉴가 안 보임** → 요금제 제한. Pro 이상 필요한 항목 있음.

## ❓ FAQ

**Q. 방문자 통계가 GA와 차이나요.**
A. 정상입니다. 아임웹·GA는 추적 방식이 달라 ±10~20% 차이 발생. 추세는 비슷해야 정상.

**Q. 통계 데이터를 엑셀로 내보낼 수 있나요?**
A. 매출은 CSV 다운로드 가능. 방문자·유입은 화면에서만 확인 (스크린샷 또는 직접 입력).

**Q. 통계가 갑자기 0으로 떨어졌어요.**
A. 사이트 일시 비공개·SSL 만료·갑작스러운 SEO 변경 등이 원인일 수 있어요. SSL 상태, 사이트 공개 여부 먼저 확인.

**Q. 광고 효율 측정에 아임웹 통계만 써도 되나요?**
A. 권장하지 않습니다. **광고 효율은 GA + 메타 픽셀**이 정확. 아임웹 통계는 보조 지표.

**Q. 방문자 통계가 6개월 후 사라져요.**
A. 요금제별로 보관 기간 다름. 장기 보관은 정기적으로 CSV 백업 또는 GA(14개월) 활용.
