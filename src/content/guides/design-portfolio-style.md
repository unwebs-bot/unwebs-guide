> 같은 "비즈니스 사이트"라도 업종마다 사용자 기대가 다릅니다. 병원 사이트가 너무 화려하면 신뢰 ↓, 카페 사이트가 너무 딱딱하면 매력 ↓. 흔한 6개 업종별 디자인 패턴·차별 포인트를 정리.

## 업종별 사용자 기대

업종 = 디자인 코드 일부 결정.

```
업종별 사용자 기대 (예상치)
─────────────────────────
병원·법무    │ 신뢰·전문성 ↑↑
기업         │ 신뢰·격식 ↑
F&B·뷰티     │ 시각적 ↑↑·식욕·매력
교육·아동    │ 활기 ↑·신뢰
테크·IT      │ 모던·미니멀
포트폴리오   │ 작품 우선·간결
```

## 1. 기업 사이트 (B2B·일반 회사)

### 목적

- 회사 신뢰도 표현
- 서비스·제품 소개
- 의뢰·견적·채용 문의 유도

### 디자인 코드

- **색**: 네이비·진회색 + 화이트 (신뢰감)
- **폰트**: 깔끔한 산세리프 (Pretendard·Noto Sans)
- **이미지**: 회사 사진·팀·실제 사례
- **헤더**: 로고 + 메뉴 + 견적 CTA
- **Hero**: 회사 정체성 한 줄 + 신뢰 신호

### 표준 구조

```
홈
├─ 회사 소개 (About)
│  ├─ 비전·미션
│  └─ 팀
├─ 서비스·제품
├─ 사례·고객사
├─ 인사이트·블로그
├─ 채용
└─ 문의
```

### 차별 포인트

- **사례** 강조 (큰 기업 로고)
- **인증·수상** 명시
- **연혁** (10년+ = 신뢰)
- **언론보도** 링크

## 2. 병원·의원 사이트

### 목적

- 진료 정보 안내
- 예약 유도
- 의료진·시설 신뢰

### 디자인 코드

- **색**: 파랑·민트 + 화이트 (청결·신뢰)
- **폰트**: 부드러운 산세리프
- **이미지**: 의료진·시설·환자 (동의 시)
- **헤더**: 로고 + 메뉴 + 예약·전화
- **Hero**: 위치 + 진료시간 + 차별 진료

### 표준 구조

```
홈
├─ 진료 안내
│  ├─ 진료과목 1
│  ├─ 진료과목 2
│  └─ 검사·시술
├─ 의료진 소개
├─ 시설·장비
├─ 진료시간·휴진
├─ 오시는 길
├─ 예약·문의
└─ 공지·건강 칼럼
```

### 차별 포인트

- **운영시간 명확** (특히 야간·휴일)
- **전화·예약 즉시 접근**
- **보험 적용 정보**
- **의료진 자격·경력**
- **LocalBusiness Schema** ([로컬 SEO](/guides/seo-local/))

> ⚠️ 의료법상 광고 제약. 환자 후기·치료 비교·강조 표현 등 신중.

## 3. 소개형·1인 사업 사이트

### 목적

- 본인·서비스 소개
- 의뢰·예약 유도
- 신뢰 형성

### 디자인 코드

- **색**: 1~2 강한 색 (개성)
- **폰트**: 디스플레이 폰트 일부 (헤딩만)
- **이미지**: 본인 사진 (꼭)
- **헤더**: 미니멀
- **Hero**: 한 줄 가치 제안 + 본인 얼굴

### 표준 구조

```
홈
├─ 소개 (본인)
├─ 서비스·강의
├─ 작업·사례
├─ 블로그
└─ 문의
```

### 차별 포인트

- **본인 사진** 크게 (얼굴이 신뢰)
- **개인 스토리**
- **사례 깊이** (양보다 질)
- **SNS·연락처** 직접 노출

## 4. F&B·식음료 사이트

### 목적

- 메뉴·분위기 소개
- 위치·예약·주문 유도
- 식욕 자극

### 디자인 코드

- **색**: 따뜻한 톤 (주황·빨강·갈색)
- **폰트**: 손글씨·디스플레이 폰트 일부
- **이미지**: 음식 사진 (대형·고해상도)
- **헤더**: 로고 + 메뉴 + 예약·주문
- **Hero**: 대표 메뉴 큰 사진 + 위치

### 표준 구조

```
홈
├─ 메뉴
├─ 매장 소개·분위기
├─ 위치·운영시간
├─ 예약·주문
└─ 공지·이벤트
```

### 차별 포인트

- **음식 사진** 압도적 ↑ (전문 촬영 권장)
- **분위기 사진** (실내·외)
- **메뉴 가격·재료** 명확
- **알레르기 정보** (필수)
- **인스타그램 연동** 갤러리

## 5. 교육·학원·아동 사이트

### 목적

- 교육 프로그램 소개
- 상담·등록 유도
- 학부모·학생 신뢰

### 디자인 코드

- **색**: 파랑·녹색 + 활기 색 (노랑·주황)
- **폰트**: 친근·둥근 산세리프
- **이미지**: 학생·교사·수업 풍경
- **헤더**: 로고 + 메뉴 + 상담 CTA
- **Hero**: 학습 가치 + 결과 (성적·합격률·작품)

### 표준 구조

```
홈
├─ 교육 프로그램
│  ├─ 과정 1
│  ├─ 과정 2
│  └─ 학년·연령별
├─ 강사진
├─ 학습 성과·후기
├─ 상담·등록
├─ 위치·시설
└─ 공지·이벤트
```

### 차별 포인트

- **합격·성적 사례**
- **학부모 후기**
- **선생님 자격·경력**
- **무료 체험·상담**

## 6. 법무·전문 서비스

### 목적

- 전문성 표현
- 상담·의뢰 유도
- 사례·실적 신뢰

### 디자인 코드

- **색**: 네이비·검정 + 골드·화이트 (전문성)
- **폰트**: 세리프 일부 (헤딩)
- **이미지**: 변호사·세무사 본인 사진·사무실
- **헤더**: 로고 + 분야 메뉴 + 상담
- **Hero**: 분야 전문성 + 무료 상담

### 표준 구조

```
홈
├─ 전문 분야
│  ├─ 분야 1 (예: 형사)
│  ├─ 분야 2 (예: 가사)
│  └─ 분야 3
├─ 변호사·전문가 소개
├─ 승소·해결 사례
├─ 칼럼·법률 정보
├─ 상담 비용·절차
└─ 무료 상담 예약
```

### 차별 포인트

- **자격·경력** 명시 (사법시험·등록번호)
- **사례** (의뢰인 익명 처리)
- **무료 1차 상담**
- **연락처 즉시 접근**

> ⚠️ 변호사법상 광고 제약. "성공 보장"·"100%" 등 금지 표현 주의.

## 공통 — 어느 업종이든

업종 무관 필수:

- **모바일 최적화**
- **HTTPS**
- **빠른 로딩** (Core Web Vitals)
- **명확한 CTA**
- **연락처 즉시 접근** (전화·이메일·카톡)
- **위치·찾아오시는 길** (오프라인 거점 있을 때)
- **About 페이지** (E-E-A-T)
- **개인정보 처리방침**

## 업종별 가격대 — 일반 기준

| 업종 | 5페이지 | 10페이지 | 복잡 |
|---|---|---|---|
| 기업 | ₩200~400만 | ₩500~1,000만 | ₩1,500만+ |
| 병원 | ₩300~500만 | ₩600~1,200만 | ₩1,500만+ |
| 1인 사업 | ₩100~300만 | ₩300~600만 | - |
| F&B | ₩200~400만 | ₩400~800만 | (주문 시스템 별도) |
| 교육 | ₩300~500만 | ₩500~1,000만 | (등록 시스템 별도) |
| 법무 | ₩300~500만 | ₩600~1,200만 | ₩1,500만+ |

지역·복잡도·디자이너 경력에 따라 변동.

## 업종 혼합 — 흔한 케이스

### 병원 + 기업 (대형 종합병원)

병원 코드 + 기업 격식. 신뢰·전문성 모두.

### 1인 사업 + 교육 (강사·코치)

소개형 + 교육 구조. 본인 사진 + 강의 안내.

### F&B + 1인 사업 (작은 카페·베이커리)

F&B 분위기 + 소개형 친근.

원칙: **주된 업종 코드 + 보조 요소**.

## ⚠️ 자주 하는 실수

- **모든 업종을 같은 톤으로**: 의료가 카페처럼·법무가 IT처럼 ❌
- **트렌드 무리 추종**: 본인 업종에 안 맞는 트렌드 X
- **참고 사이트 카피**: 영감 OK, 그대로 복사 X
- **본인 사진 빠짐 (1인 사업)**: 가장 중요한 신뢰 신호 누락
- **음식 사진 작게 (F&B)**: 핵심 시각 자산 무시

## ❓ FAQ

**Q. 우리 업종은 위에 없어요. 어떡하나?**
A. 가장 비슷한 업종 + 가까운 업종 혼합. 또는 같은 업종의 1순위 사이트 5개 분석.

**Q. 트렌디한 디자인 vs 안전한 디자인?**
A. 업종에 따라. 의료·법무는 안전. 패션·디자인 회사는 트렌디.

**Q. 색·폰트를 업종 코드 무시하고 본인 취향대로?**
A. 가능. 다만 "왜 이 업종이 이런 색을 쓰는가" 이유는 인지. 의도적 차별이 아니면 표준이 안전.

**Q. 디자인 수정 빈도?**
A. 처음 디자인 1~2년 유지. 그 후 리뉴얼 검토. 너무 자주 ↔ 너무 옛 ↔ 균형.

**Q. AI 디자인 도구 활용?**
A. 시안 영감용 OK. 그대로 사용 X. 업종 코드 이해 + 본인 사이트 맞춤 필수.
