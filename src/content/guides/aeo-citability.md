> AI 답변 엔진은 글 전체를 인용하지 않습니다. **잘라서 그대로 따 갈 수 있는 조각(citable chunk)** 만 가져가요. "AI 엔진은 데이터·전문성으로 뒷받침된 명확하고 구체적인 주장을 찾는다. 모호하고 뜬구름 잡는 단락은 건너뛴다." 이 글은 인용되는 글의 **구조**를 다룹니다.
>
> 어떤 사이트가 인용되는지(신뢰·권위)는 [AEO/GEO란?](/guides/aeo-fundamentals/), 어떤 상황을 노릴지는 [CEP](/guides/aeo-cep/)에서 다룹니다. 여기선 *문장·단락 구조*에 집중합니다.

## Citability란?

AI가 답변을 만들 때 **출처에서 잘라 붙일 수 있는 정도**입니다. 같은 정보라도 어떻게 구조화했느냐에 따라 인용 확률이 크게 달라져요.

```
❌ 모호한 단락 (스킵됨)
"홈페이지 제작 기간은 여러 요인에 따라 다양하게 달라질 수 있으며,
프로젝트의 성격과 규모를 종합적으로 고려해야 합니다."

✅ 인용 가능한 문장
"일반 소개형 홈페이지 제작은 보통 4주가 걸린다.
페이지 5개 기준, 콘텐츠가 준비돼 있으면 3주로 단축된다."
```

→ 두 번째는 **명시적 주장 + 구체 수치**라 AI가 그대로 따 갑니다.

## 인용되는 6가지 구조 패턴

| 패턴 | 설명 | 예시 |
|---|---|---|
| **명시적 정의문** | "X는 ~이다" 한 문장에 정의 압축 | "Core Web Vitals는 구글이 쓰는 3가지 사이트 품질 지표다." |
| **구체 수치·연도** | 측정값·기간·비율 | "LCP는 2.5초 이내가 양호 기준이다." |
| **단계별 프로세스** | 번호 매긴 절차 | "1. 도메인 연결 2. SSL 설치 3. ..." |
| **비교표** | 2~3개 옵션의 명시적 차이 | "워드프레스 vs 아임웹" 표 |
| **Q·A 구조** | 질문 → 짧은 답 | FAQ 섹션, H2를 질문으로 |
| **외부 권위 인용** | 공신력 있는 출처 | "구글 공식 문서에 따르면..." |

> 💡 핵심: **각 조각이 단독으로 읽혀도 말이 되게.** 앞 문맥을 알아야만 이해되는 단락은 잘려 나가면 의미를 잃어 인용이 안 됩니다.

## 글 쓸 때 적용하는 법

### 1. 결론을 먼저, 한 문장으로
각 섹션 첫 문장에 핵심 답을 박습니다. AI도 사람도 그 문장을 먼저 봅니다.

### 2. 헤딩(H2·H3)을 질문으로
```
✅ "홈페이지 제작 비용은 얼마인가요?"
✅ "SSL은 꼭 설치해야 하나요?"
```
사용자가 AI에 던지는 질문과 헤딩이 일치하면 인용 확률이 오릅니다.

### 3. FAQ를 3~5쌍 넣기
질문 → 짧고 완결된 답. AI가 가장 인용하기 좋은 구조입니다. ([FAQ schema](/guides/aeo-faq-schema/)도 함께)

### 4. 숫자로 말하기
"많은", "대부분" → "10건 중 7건", "월 1,200명". 막연한 형용사를 측정값으로.

### 5. 비교는 표로
"A는 ~하고 B는 ~하다"를 산문으로 풀지 말고 표로. AI가 구조를 그대로 읽습니다.

## non-commodity — AI도 쓸 수 있는 글은 안 인용된다

2026년 구글이 가장 강조한 개념입니다. AI가 이미 알고 있는 일반 지식은 인용할 이유가 없어요.

| Commodity (인용 안 됨) | Non-commodity (인용됨) |
|---|---|
| "홈페이지 제작 팁 7가지" | "이 조건에서 제작비가 2배가 된 이유" |
| 일반 지식 나열 | 실제 경험·수치·관찰 |
| 누구나 30초에 쓸 수 있음 | 직접 해보지 않으면 못 씀 |

> ⚠️ Citability 구조를 갖춰도 **내용이 일반 지식이면** 인용 안 됩니다. 구조(이 글) + 고유 인사이트([E-E-A-T](/guides/seo-eeat/)) 둘 다 필요해요.

## 기술 트릭은 본질이 아니다 — 균형

일부 자료는 "schema·chunking·llms.txt로 AI 신호를 준다"고 합니다. 하지만 **구글 공식은 이들이 생성형 AI 검색에 필수가 아니라고** 명시했어요.

- ✅ **본질** (양쪽 동의): 구체적 주장 + 데이터 + 토픽 권위 + 인용 가능한 구조
- 🛡️ **보험**: 비용 낮은 FAQ·Organization schema는 유지 (구글 외 AI엔 도움 가능)
- ❌ **과투자 금물**: 인위적 chunking, llms.txt, schema 도배는 ROI 낮음

> 정리하면, **인용은 트릭이 아니라 "잘 구조화된 유용한 글"의 부산물**입니다.

## 인용 가능성 체크리스트 (글 발행 전)

- [ ] 각 섹션 첫 문장에 핵심 답이 있는가
- [ ] H2·H3가 사용자 질문 형태인가
- [ ] 명시적 정의문이 1개 이상 있는가
- [ ] 구체 수치·연도가 들어갔는가
- [ ] 비교는 표로 정리했는가
- [ ] FAQ 3~5쌍이 있는가
- [ ] 외부 권위 인용이 1~3건 있는가
- [ ] 각 단락이 단독으로 읽혀도 말이 되는가

## ⚠️ 자주 하는 실수

- **뜬구름 단락**: "다양한 요인에 따라 달라집니다" → AI가 스킵
- **결론을 맨 뒤에**: 핵심을 끝에 두면 잘려 나감
- **헤딩이 명사구**: "비용 안내"보다 "비용은 얼마인가요?"
- **막연한 형용사**: "많은", "대부분" → 수치로
- **구조만 갖추고 내용은 일반 지식**: commodity는 인용 안 됨
- **기술 트릭 과투자**: chunking·llms.txt에 시간 쏟기

## ❓ FAQ

**Q. FAQ를 넣으면 정말 인용이 잘 되나요?**
A. Q·A 구조는 AI가 "질문-답" 쌍으로 인식하기 가장 쉬운 형태입니다. 사용자가 AI에 던지는 질문과 FAQ 질문이 비슷하면 그 답이 인용될 확률이 높아져요.

**Q. schema(FAQ schema 등)를 꼭 넣어야 인용되나요?**
A. 구글 공식은 "필수 아니다"라고 합니다. 다만 비용이 낮으니 보험으로 넣되, schema만으로 인용을 기대하진 마세요. **본문 구조와 내용이 우선**입니다.

**Q. 글을 짧은 조각으로 잘라 써야 하나요?**
A. 인위적으로 잘게 쪼갤 필요는 없습니다(구글도 불필요하다고 명시). 자연스러운 글에서 **각 단락이 명확하고 단독으로 읽히게** 쓰면 충분합니다.

**Q. 어떤 AI 엔진에 맞춰야 하나요?**
A. 특정 엔진 하나에 최적화하지 마세요. 명확한 구조 + 고유 내용은 ChatGPT·Perplexity·AI Overviews·네이버 큐 모두에 부분적으로 효과적입니다.
