> 글을 아무리 많이 써도 흩어져 있으면 검색 권위가 안 쌓입니다. **한 주제를 허브 글 1편 + 세부 글 여러 편 + 내부 링크로 엮으면** 검색엔진이 "이 사이트는 이 주제의 권위자"로 인식해요. 이게 토픽 클러스터(topic cluster)입니다. AI 검색도 토픽 권위가 강한 출처를 인용합니다.

## 3가지 구성 요소

| 요소 | 정의 | 예시 |
|---|---|---|
| **Pillar (허브)** | 주제를 넓게 다루는 대표 글 1편. 모든 세부 글로 링크 | "기업 홈페이지 제작 완벽 가이드" |
| **Cluster (스포크)** | 각자 구체적 하위 주제·키워드를 잡는 글 다수 | "치과 홈페이지 제작", "학원 홈페이지 제작" 등 |
| **내부 링크** | 허브 ↔ 세부, 세부 ↔ 세부 양방향 연결 | 허브가 세부 5개로, 각 세부가 허브로, 관련 세부끼리 |

> 💡 비유하면 **지하철 노선도**입니다. 중앙역(허브)에서 각 역(세부)으로 노선(내부 링크)이 뻗어 있는 구조예요.

## 왜 효과적인가

### 1. 토픽 권위(topical authority)
한 주제를 여러 각도로 깊이 다루면 검색엔진과 사용자 모두 "권위 있는 출처"로 인식합니다. AI 답변 엔진도 이런 출처를 우선 인용해요.

### 2. 다중 키워드 노출
세부 글마다 다른 키워드를 잡으니 사이트 전체의 검색 키워드가 다양해집니다.

### 3. 다중 의도 흡수
허브는 넓은 의도를, 세부는 구체적 의도를 잡습니다. ([검색 의도](/guides/seo-search-intent/) 참고)

### 4. 키워드 잠식(카니발) 회피
메인 키워드 하나당 허브 1편만 두고, 세부는 롱테일 변형을 잡으면 **내 글끼리 같은 키워드로 경쟁하는 일이 없어집니다.**

### 5. 생산 효율
어디에 어떤 글을 쓸지 미리 정해져 있어 우선순위가 명확합니다.

## Pillar 페이지 만드는 3가지 방식

| 방식 | 형식 | 추천도 |
|---|---|---|
| **챕터 인덱스** | 소개 + 챕터 목록 → 각 세부 글로 링크 | ⭐ 깔끔, 기본값 추천 |
| **탭 네비게이션** | 상단 탭으로 세부 주제 전환 | 깊이 있는 사이트에 적합 |
| **통합 가이드 + 목차** | 한 페이지에 모든 섹션 + 인라인 링크 | 길지만 강력 |

## 클러스터 만드는 6단계

### 1. 주제 리스트부터 (키워드 X)
키워드가 아니라 **권위를 세우고 싶은 주제**부터 적습니다. 고객의 고민 ↔ 우리 해법을 매핑하세요.

### 2. 키워드 발굴
주제별로 모든 변형과 롱테일을 모읍니다. 도구: 네이버 키워드도구, 블랙키위, Google Keyword Planner. [검색 의도](/guides/seo-search-intent/)·검색량·난이도로 거릅니다.

### 3. 허브·세부 문서화
스프레드시트나 다이어그램으로 구조를 시각화합니다. 어떤 글이 허브이고 어떤 글이 세부인지 한눈에.

### 4. 글 작성 — 세부는 키워드 1개만
**각 세부 글은 메인 키워드 1개만** 잡습니다. 상위 SERP를 분석해 더 나은 글을 씁니다.

### 5. 내부 링크 구축
- 허브 → 모든 세부 글
- 각 세부 글 → 허브
- 관련 있는 세부 글끼리 → 가로 연결
- **앵커 텍스트는 서술형**으로: "치과 홈페이지 제작 가이드" (O) / "여기 클릭" (X)

### 6. 측정·반복
발행 후 4주간 모니터링. 부진한 세부 글은 보강, 잘 되는 글은 확장합니다. ([Search Console 측정](/guides/seo-search-console/) 참고)

## 좋은 앵커 텍스트 vs 나쁜 앵커 텍스트

```
✅ "워드프레스 보안 강화 방법은 별도 가이드에서 정리했습니다"
   (링크 텍스트 = "워드프레스 보안 강화 방법")

❌ "자세한 내용은 여기를 클릭하세요"
   (링크 텍스트 = "여기")
```

검색엔진은 앵커 텍스트로 링크된 페이지의 주제를 파악합니다. "여기 클릭"은 아무 정보도 주지 않아요.

## ⚠️ 자주 하는 실수

- **키워드부터 시작**: 주제부터 시작해야 권위가 쌓임
- **허브 없이 세부 글만 발행**: 흩어진 글은 토픽 권위가 안 생김
- **같은 메인 키워드를 노리는 세부 글 2개**: 서로 잠식
- **의미 없는 앵커 텍스트**: "여기", "더 보기"
- **허브 → 세부 한 방향만**: 양방향 + 가로 연결이 핵심
- **세부 글 하나에 키워드 2개 이상**: 하나만 집중

## ❓ FAQ

**Q. 글이 몇 편 있어야 클러스터가 되나요?**
A. 허브 1편 + 세부 5~7편이 표준입니다. 적어도 허브 1 + 세부 3편은 있어야 구조가 작동해요. 처음엔 작게 시작해 확장하면 됩니다.

**Q. 허브 글은 얼마나 길어야 하나요?**
A. 주제를 넓게 포괄해야 하므로 보통 3,000자 이상, 길면 5,000자 이상입니다. 세부 글은 1,500~3,000자면 충분해요.

**Q. 기존에 흩어진 글들을 클러스터로 묶을 수 있나요?**
A. 가능합니다. 관련 글을 모아 허브 글을 새로 쓰고, 기존 글들과 내부 링크로 연결하면 됩니다. 이미 있는 자산을 살리는 좋은 방법이에요.

**Q. 내부 링크를 너무 많이 걸면 스팸 아닌가요?**
A. 관련성이 있으면 문제없습니다. 억지로 무관한 글을 연결하는 게 문제예요. 독자에게 실제로 도움 되는 링크만 거세요.
