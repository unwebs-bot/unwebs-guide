> 아임웹 사이트를 인수받았거나 처음 만들었다면 **이 순서로** 진행하세요: 관리자 접속 → 기본 정보 입력 → 약관 설정 → 대표 이미지 → 미리보기 → 게시. 30분이면 기본 골격이 완성됩니다.

## 시작하기 전에

- **계정 권한 확인**: 사이트 소유자 또는 운영진 권한 필수.
- **준비물**: 사이트 이름·설명·로고·대표 이미지(1200×630px PNG).
- **소요 시간**: 30~60분.

## 1. 관리자 페이지 접속하기

### 방법 A: 아임웹 홈에서

1. [imweb.me](https://imweb.me)에 접속하고 로그인.
2. 우측 상단 **내사이트** 클릭.
3. 해당 사이트의 **사이트 관리** 버튼 → 관리자 페이지 진입.

### 방법 B: 빠른 접속 (운영진은 이 방법만 가능)

브라우저 주소창에 `사이트주소/admin` 입력 → 로그인.

![관리자 페이지 접속 화면](/imweb/528a5ee752fc9.png)

> 💡 운영진(공동 관리자) 계정은 아임웹 홈에서 사이트가 보이지 않습니다. **반드시 `/admin` 직접 접속**이 필요해요.

## 2. 사이트 이름·설명 입력 (SEO 직결)

**관리자 > 환경설정 > 일반**에서 설정.

### 사이트 이름

| 좋은 예 | 안 좋은 예 |
|---|---|
| 편한이비인후과 - 안산 본오동 | 우리 병원 |
| 언웹스 - 1인 웹팩토리 | 회사 홈페이지 |
| 책방 BookLab | 서점 |

> 일반 명사("회사", "병원")로만 짓지 마세요. 검색엔진이 일반 정보 검색으로 인식해 상위 노출이 어려워집니다.

### 사이트 설명 (가장 중요)

**60~80자**. 검색 결과에 그대로 노출되므로 신중히.

- 누구를 위한 사이트인지
- 어떤 가치·서비스를 제공하는지
- 지역·차별점 (있다면)

예시:
> 안산 본오동 편한이비인후과는 코·귀·목 전문 진료를 제공합니다. 평일 야간 9시까지, 토요일 진료. 수면다원검사·알레르기 비염 클리닉 운영.

자세한 SEO 최적화는 [imweb-seo-basics](/dev/imweb/imweb-seo-basics/) 참고.

## 3. 파비콘·대표 이미지 등록

같은 화면(**환경설정 > 일반**)에서.

### 파비콘

브라우저 탭에 표시되는 작은 아이콘. ICO 형식 권장 (PNG도 가능).

### 대표 이미지 (Open Graph)

카카오톡·페이스북에 사이트 공유 시 함께 표시되는 이미지.

- **크기**: 1200×630px (Open Graph 표준)
- **형식**: PNG만 가능 (아임웹 제약)
- **용량**: 8MB 이하
- **내용**: 로고 + 사이트명 + 한 줄 카피 정도

![대표 이미지 업로드 화면](/imweb/372534e6e3e7f.png)

> ⚠️ 카카오톡에서 대표 이미지가 안 바뀌면 카카오톡 캐시 문제입니다. 카카오톡 → 설정 → 캐시 데이터 삭제 후 재공유.

## 4. 약관 설정 (법적으로 필수)

회원가입·문의폼·결제 중 하나라도 사용하면 약관이 **법적으로 필수**.

### 위치

**관리자 > 환경설정 > 약관**

### 필수 약관 3종

| 약관 | 언제 필요? | 비고 |
|---|---|---|
| 이용약관 | 회원가입·결제 시 | 아임웹 기본 템플릿 활용 가능 |
| 개인정보처리방침 | **모든 사이트 필수** | 개인정보 1건이라도 수집 시 |
| 개인정보 제3자 제공동의 | 외부 업체에 정보 전달 시 | 예: 굿스플로 배송, 결제 PG |

> 💡 변호사 검토 없이 약관을 쓰는 건 위험할 수 있습니다. 아임웹 템플릿은 출발점일 뿐, **사업 특성에 맞게 수정** 필수.

## 5. 운영진 추가하기

혼자 운영하지 않으면 추가 관리자(운영진)를 두는 게 안전합니다.

### 위치

**관리자 > 사용자 관리 > 운영진**

### 추가 절차

1. **운영진 추가** 버튼.
2. 운영진의 이메일 입력 → 권한 그룹 선택.
3. 운영진에게 초대 메일 발송.
4. 운영진은 메일의 링크로 가입 → `사이트주소/admin`으로 로그인.

### 권한 그룹 분리 권장

- **마스터 관리자**: 본인 1명. 모든 권한.
- **콘텐츠 관리자**: 글·이미지만 수정. 결제·회원 관리 X.
- **운영팀**: 주문·문의 응대만. 디자인 수정 X.

> 운영진별로 개별 계정을 발급하세요. **하나의 계정을 공유하면** 누가 무엇을 했는지 추적이 안 됩니다.

## 6. 미리보기·게시 — 작업 끝나면

아임웹은 **자동 저장**되지만, **방문자에게 공개되는 건 "게시하기"**부터입니다.

### 작업 흐름

```
디자인 모드에서 수정 (자동 저장)  →  미리보기로 확인  →  게시하기  →  방문자에게 노출
```

### 미리보기

- 디자인 모드 우측 상단 **미리보기** 아이콘.
- 관리자만 보이는 비공개 미리보기. 코드 위젯·롤오버·모달 등은 미리보기에서만 작동.
- **시크릿 브라우저 창**으로 미리보기 URL을 열면 비로그인 방문자 입장에서 볼 수 있어요.

### 게시하기

⚠️ **한 번 게시하면 되돌릴 수 없습니다.** 미리보기로 충분히 확인 후 게시.

### 복원하기 (긴급용)

게시 후 문제가 발견되면 **마지막 게시 시점으로 복원** 가능. 단, 복원 후엔 **현재 상태로 다시 못 돌아옵니다**. 신중히.

## 보너스: 사이트 작업 중 일시적으로 가리기

리뉴얼이나 큰 수정 중에 방문자에게 어수선한 화면을 보이고 싶지 않다면:

### 방법 1: 전체 비공개

**환경설정 > 일반 > 공개 범위 > 비공개**.
방문자 전원 접근 불가.

### 방법 2: 공사중 페이지만 보이기

신규 메뉴를 만들어 "공사중" 화면으로 사용. 자세한 절차는 [imweb-renewal-mode](/dev/imweb/imweb-renewal-mode/) 참고.

## ⚠️ 자주 막히는 포인트

- **"왜 변경한 내용이 사이트에 안 보여요?"** → **게시하기**를 누르지 않으면 방문자에게 안 보입니다.
- **"내 사이트가 안 떠요"** → 운영진 계정으로는 `/admin` 직접 접속만 가능.
- **약관을 영작·복붙으로 채움** → 법적 효력 없을 수 있음. 사업 특성에 맞게 수정 필수.
- **모든 권한을 한 사람에게 몰아줌** → 인수인계 시 위험. 권한 그룹 분리.

## ❓ FAQ

**Q. 무료 버전(Free)으로 시작해도 되나요?**
A. 가능하지만 **개인 도메인 연결 불가**, 메뉴 5개 제한, 광고 표시 등 제약이 큽니다. 사업·기관 사이트는 **Starter 이상** 권장.

**Q. 도메인을 아직 안 샀어요.**
A. `mysite.imweb.me` 같은 기본 주소로 먼저 만들고 나중에 개인 도메인을 연결해도 됩니다. 다만 검색엔진 색인은 개인 도메인 연결 후가 깔끔합니다. [imweb-domain-connect](/dev/imweb/imweb-domain-connect/) 참고.

**Q. 게시한 후 마음에 안 들면 어떻게 하나요?**
A. 게시 전 상태로 되돌리는 **복원** 기능이 있습니다. 단, 복원 후엔 "복원 전"으로 다시 못 돌아갑니다. 게시 전 반드시 미리보기로 확인.

**Q. 운영진 권한 그룹은 몇 개까지 만들 수 있어요?**
A. 요금제에 따라 다르지만 보통 충분합니다. 단, 권한이 많을수록 관리 복잡도가 늘어요. 3~4개 정도가 적정.

**Q. 첫 게시 후 검색엔진에 바로 노출되나요?**
A. 아닙니다. 네이버 약 2주, 구글 약 2~4주 소요. [imweb-seo-basics](/dev/imweb/imweb-seo-basics/) 가이드로 등록·최적화를 미리 해두세요.
