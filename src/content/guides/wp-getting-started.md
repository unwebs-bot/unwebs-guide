> WordPress는 **전 세계 웹사이트의 43%가 사용하는** 콘텐츠 관리 시스템(CMS)입니다. 무료·오픈소스이고 확장성·자유도가 가장 높아요. 본 가이드는 처음 WordPress를 운영하게 된 분을 위한 입문 — 무엇이고, 어떻게 시작하고, 무엇부터 챙겨야 하는지.

## WordPress란?

웹사이트를 만들고 관리하는 **무료 오픈소스 소프트웨어**. 두 가지 형태가 있어 혼동 주의:

| WordPress.com | WordPress.org |
|---|---|
| 호스팅 포함 SaaS | 본인 서버 설치 |
| 무료 ~ 유료 플랜 | 완전 무료 + 호스팅 비용 |
| 기능·테마 제한 | 모든 플러그인·테마 자유 |
| 기업·전문가 추천 X | **기업·전문가 표준** |

본 가이드는 **WordPress.org (자체 호스팅)** 기준.

## WordPress를 쓰는 이유

### 강점

- **무료** (코어·대부분 플러그인)
- **확장성** — 60,000+ 플러그인 + 11,000+ 테마
- **SEO 강함** — Yoast·Rank Math 등 SEO 플러그인
- **콘텐츠 운영 편함** — 글·페이지·미디어 통합 관리
- **소유권** — 본인 호스팅이라 데이터 100% 본인 것
- **커뮤니티** — 전 세계 가장 큰 CMS 생태계

### 약점

- **자체 관리 부담** — 보안 업데이트·백업·서버 운영
- **느려질 수 있음** — 플러그인 많이 설치 시
- **취약점** — 가장 많이 쓰이는 만큼 해킹 시도 많음
- **호스팅 비용** — 월 $5~50+

## 시작 — 3가지 옵션

### 옵션 1: 일반 호스팅 + WordPress

가장 흔한 선택. 호스팅사가 자동 설치 제공.

추천 호스팅:
- **국내**: 카페24·가비아·후이즈 (한국어 지원)
- **글로벌**: Bluehost·SiteGround·DreamHost
- **관리형 WP**: Kinsta·WP Engine (속도·보안 ↑, 가격 ↑)

가격: 월 $5 ~ $40.

### 옵션 2: Local (개발용)

본인 컴퓨터에 설치. 학습·개발용. 실제 운영 X.

- [Local by Flywheel](https://localwp.com) — 가장 쉬움 (Mac·Windows)
- MAMP·XAMPP

### 옵션 3: 클라우드 (고급)

AWS·Vultr·DigitalOcean 등. 직접 서버 운영. 비용 절감·자유도 ↑, 기술 부담 ↑.

> 💡 **시작은 옵션 1** (일반 호스팅). 성장하면 옵션 3 고려.

## 설치 절차 (일반 호스팅 기준)

### 1. 호스팅 가입 + 도메인 연결

호스팅사 가입 → 도메인 구입(또는 기존 도메인 연결).

### 2. WordPress 자동 설치

대부분 호스팅사가 "1-Click WordPress Install" 제공:

1. 호스팅 cPanel 또는 관리 페이지 접속
2. **WordPress 자동 설치** 클릭
3. 사이트 정보 입력 (제목·관리자 ID·비밀번호)
4. 설치 완료 (보통 1~3분)

### 3. 접속

```
사이트: https://mycompany.com
관리자: https://mycompany.com/wp-admin
```

`/wp-admin`으로 접속해 관리자 ID·비밀번호 로그인.

### 4. 초기 설정

- **설정 > 일반**: 사이트 제목·태그라인·시간대
- **설정 > 읽기**: 첫 페이지를 "최신 글" or "고정 페이지" 중 선택
- **설정 > 고유주소**: **"게시물 이름"** 선택 (SEO에 필수)

## 첫 페이지·글 발행

### 페이지 (정적 — About, Contact)

1. 좌측 메뉴 **페이지 > 새로 추가**
2. 제목 입력 → 본문 작성 (Gutenberg 블록 에디터)
3. 오른쪽 패널에서 발행 상태 확인
4. **공개** 버튼 → 공개됨

### 글 (동적 — 블로그 포스트)

1. 좌측 메뉴 **글 > 새로 추가**
2. 제목·본문 작성
3. **카테고리** 선택, **태그** 입력
4. **대표 이미지** 설정 (SNS 공유 시 표시)
5. **공개**

## 30분 안에 끝낼 첫 설정 체크리스트

- [ ] **고유주소 설정** (`설정 > 고유주소 > 게시물 이름`)
- [ ] **시간대** 한국 (`설정 > 일반 > 시간대 > Seoul`)
- [ ] **사이트 제목·태그라인** 설정 (SEO 영향)
- [ ] **기본 카테고리** 변경 (`Uncategorized` → 본인 첫 카테고리)
- [ ] **불필요한 샘플 콘텐츠 삭제** (Hello World 글·샘플 페이지)
- [ ] **댓글 설정** (`설정 > 토론` — 스팸 방지 옵션)
- [ ] **관리자 비밀번호 강화** (특수문자 + 12자+)
- [ ] **테마 결정·설치** (테마 가이드 참고)
- [ ] **필수 플러그인 5개 설치** (보안·백업·SEO·캐시·폼)

자세한 단계는 다음 가이드들 참고.

## 입문자에게 가장 흔한 질문

### "테마는 무엇을 골라야?"

- 사용자: **Astra·GeneratePress·Kadence** (가벼움)
- 디자인 직접 만들기: **자식 테마** ([테마 설치·전환](/guides/wp-theme-install/) 참고)
- 일단 시작: WordPress 기본 테마 (Twenty Twenty-Five)

### "플러그인은 몇 개나 설치?"

- **5~15개 적정**
- 30개 넘으면 속도·충돌 위험 ↑
- 필수 카테고리만: 보안·백업·SEO·캐시·폼

### "유료 vs 무료 플러그인?"

- **시작은 무료**로 충분
- 비즈니스 사이트라면 핵심 1~2개 유료(예: Rank Math Pro, WP Rocket)
- "Bundle 할인" 사기 주의

### "WordPress 5 vs 6 차이?"

- 현재는 6.x. **블록 에디터(Gutenberg)** 기반
- 옛 Classic Editor 쓰던 분은 Gutenberg 적응 기간 필요

## ⚠️ 초보자가 자주 하는 실수

- **백업 안 함**: 첫 사고 시 모든 데이터 소실
- **`admin` ID 사용**: 해킹 시도 1순위 대상
- **약한 비밀번호**: 6자 일반 단어
- **WordPress·플러그인 업데이트 안 함**: 보안 취약점 방치

## ❓ FAQ

**Q. 코드 몰라도 WordPress 운영 가능한가요?**
A. 운영 자체는 GUI 위주라 코딩 불필요. 다만 **고급 커스터마이징·문제해결**은 HTML·CSS·PHP 기초 도움 됨.

**Q. WordPress와 아임웹 중 뭐가 좋아요?**
A. **WordPress**: 자유도·확장성·소유권 ↑, 자체 관리 부담 ↑. **아임웹**: 시작 쉬움·자체 관리 X, 자유도 제약. 사이트 규모·기술 수준에 따라.

**Q. WordPress 사용료가 무료 맞나요?**
A. 소프트웨어는 무료. 다만 **호스팅 비용** (월 $5~$50) + 도메인 ($10~$30/년) + 유료 테마·플러그인(선택)이 실제 비용.

**Q. WordPress 사이트는 보안이 약한가요?**
A. WordPress 코어 자체는 안전. 다만 **인기 = 해킹 시도 1위**. 정기 업데이트·보안 플러그인·강한 비밀번호로 90% 해결.

**Q. 무료 테마·플러그인이 안전한가요?**
A. **공식 저장소**(wordpress.org)의 무료는 검수 완료라 안전. 외부 사이트의 "프리미엄 nulled" 같은 불법 무료는 악성코드 위험.
