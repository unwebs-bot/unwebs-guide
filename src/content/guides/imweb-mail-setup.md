> 회원가입·주문·문의 폼 접수 시 자동 메일이 발송되려면 **메일 서비스 제공업체**를 먼저 지정해야 합니다. 네이버·Gmail·다음·네이트 또는 개인 도메인 SMTP. 이 글에서는 각 방식의 설정 절차와 자주 발생하는 문제를 정리.

## 시작하기 전에

- 어떤 이메일 주소를 발송용으로 쓸지 결정 (예: `info@mycompany.com`).
- 해당 메일 계정의 비밀번호 또는 앱 비밀번호 준비.
- **소요 시간**: 10~20분.

---

## 어떤 메일 서비스를 골라야 할까?

| 메일 서비스 | 추천 | 비고 |
|---|---|---|
| **Gmail** | ⭐ 가장 안정적 | 앱 비밀번호 필요. 발송 한도 일 500건 |
| **네이버** | ⭐ 한국 사용자에 친숙 | 외부 앱 비밀번호 별도 필요 |
| **개인 도메인 SMTP** | ⭐ 브랜드 신뢰도 ↑ | `info@mycompany.com` 형태로 발송 |
| **다음·네이트** | 가능은 함 | 발송 안정성 보장 어려움 |
| **메일건(Mailgun)** | 대량 발송 | 유료 외부 SMTP 서비스 |

> 💡 **회사 사이트는 개인 도메인 SMTP 가장 권장**. `mycompany.com` 도메인으로 가입돼 있어야 신뢰도가 높아요.

## 1. Gmail로 설정하기

### 사전 준비: 앱 비밀번호 발급

Google 계정의 일반 비밀번호로는 외부 앱(아임웹) 연결 불가. **앱 비밀번호** 별도 발급 필요.

1. [Google 계정 보안 설정](https://myaccount.google.com/security) 접속.
2. **2단계 인증** 활성화 (필수).
3. 아래로 스크롤 → **앱 비밀번호** 클릭.
4. 앱 이름 입력 (예: `아임웹`) → **만들기**.
5. 16자리 비밀번호 표시됨 → 메모장에 복사.

### 아임웹 설정

1. **환경설정 > 자동 메일** → **메일 제공업체** 클릭.
2. **Gmail** 선택.
3. 입력:
   - **이메일 주소**: Gmail 주소
   - **비밀번호**: 위에서 발급받은 **앱 비밀번호** (Google 계정 비밀번호 아님)
4. **저장**.

### 발송 테스트

1. 사이트 입력폼 또는 회원가입 시도.
2. 작성자에게 자동 메일 수신 확인.
3. 안 오면 스팸함 확인.

## 2. 네이버로 설정하기

### 사전 준비: 외부 앱 비밀번호 발급

1. 네이버 → 내정보 → **보안설정** → **2단계 인증** 활성화.
2. **외부 앱 비밀번호 관리** 클릭.
3. **새 외부 앱 비밀번호 발급** → 앱 이름 (예: `아임웹`) → **생성**.
4. 발급된 비밀번호 복사.

### 아임웹 설정

1. **메일 제공업체** → **네이버**.
2. 입력:
   - **이메일 주소**: 네이버 메일 주소 (`xxx@naver.com`)
   - **비밀번호**: 위에서 발급받은 **외부 앱 비밀번호**
3. **저장**.

> ⚠️ 네이버 일반 비밀번호로 시도 시 "인증 실패". 반드시 외부 앱 비밀번호 사용.

## 3. 개인 도메인 SMTP 설정 (가장 추천)

`info@mycompany.com` 형식의 메일을 발송 주소로 쓰려면 SMTP 직접 설정.

### 사전 준비

- 개인 도메인 연결 완료 ([도메인 연결 가이드](#) 참고).
- MX 레코드 설정 완료 (메일 서비스 사용 시).
- 메일 서버 정보 (SMTP 호스트·포트·계정):
  - Google Workspace: `smtp.gmail.com:587`
  - 다음 스마트워크: `smtp.daum.net:465`
  - 네이버웍스: `smtp.worksmobile.com:587`
  - 호스팅사 자체 SMTP: 호스팅사에 문의

### 아임웹 설정

1. **메일 제공업체** → **사용자 지정 SMTP** 선택.
2. 입력:
   - **SMTP 서버 주소**: 예 `smtp.gmail.com`
   - **포트**: 보통 465 (SSL) 또는 587 (TLS)
   - **이메일 주소**: 발송용 메일 (`info@mycompany.com`)
   - **비밀번호**: 해당 메일 비밀번호 (Google Workspace는 앱 비밀번호)
   - **암호화 방식**: SSL 또는 TLS
3. **저장**.

### Google Workspace SMTP 예시

| 항목 | 값 |
|---|---|
| SMTP 호스트 | `smtp.gmail.com` |
| 포트 | `587` |
| 암호화 | TLS |
| 사용자명 | `info@mycompany.com` |
| 비밀번호 | 앱 비밀번호 |

> 💡 Google Workspace를 쓰면 **회사 도메인 메일**(`info@mycompany.com`) + **Gmail 사용성**을 모두 얻습니다. 월 6,800원~ 부담되지 않는 수준.

## 발송되는 자동 메일 종류

아임웹이 자동으로 보내는 메일:

| 메일 | 발송 시점 |
|---|---|
| 회원가입 환영 | 신규 가입 |
| 비밀번호 찾기 | 사용자 요청 |
| 입력폼 접수 알림 | 폼 제출 시 |
| 주문 알림 | 주문 완료 시 |
| 게시판 댓글 알림 | 댓글 작성 시 |
| 관리자 알림 | 위 모든 이벤트 시 |

### 자동 메일 내용 수정

**환경설정 > 자동 메일**에서 메일 종류별 제목·본문 편집 가능. 변수(`{이름}`, `{사이트명}` 등)도 사용 가능.

## 발송 안 될 때 체크리스트

### 1. 메일 제공업체 비밀번호 오류
가장 흔한 원인. **일반 비밀번호 vs 앱 비밀번호** 헷갈리지 않게.

### 2. 스팸함 분류
받는 이의 스팸함 확인. 다음 항목으로 스팸 분류 방지:
- 발신자명을 한글이 아닌 사이트명으로
- 메일 본문에 ALL CAPS·과도한 이모지 X
- 첨부 파일 없이 발송
- 발신자 도메인의 SPF·DKIM 레코드 설정

### 3. 도메인 SMTP 인증 실패
- DNS의 SPF 레코드에 SMTP 서버 등록
- 발신자 도메인과 SMTP 서버 도메인이 일치하지 않으면 다수의 메일 서비스가 차단

### 4. 메일 발송 한도 초과
- Gmail: 일 500건
- Google Workspace: 일 2,000건
- 사용자 지정 SMTP: 호스팅사 정책

## ⚠️ 자주 막히는 포인트

- **일반 비밀번호로 시도해서 인증 실패** → Gmail·네이버는 **앱 비밀번호** 필수.
- **자동 메일이 스팸함으로** → SPF·DKIM 미설정 도메인은 스팸으로 자주 분류. 도메인 호스팅처에서 SPF 추가.
- **`@imweb.me` 발신자로 발송됨** → 메일 제공업체 미설정 상태. **반드시 본인 도메인 메일로 설정**.
- **자동 메일 내용을 본인이 못 바꿈** → 일부 시스템 메일(비밀번호 찾기 등)은 변수만 수정 가능, 본문은 제한적.

## ❓ FAQ

**Q. 네이버 외부 앱 비밀번호가 어디 있는지 모르겠어요.**
A. 네이버 로그인 → 내정보 → **보안설정** → **2단계 인증** → **외부 앱 비밀번호 관리** → **새 외부 앱 비밀번호 발급**.

**Q. 자동 메일 발송이 갑자기 안 됩니다.**
A. 다음을 점검:
1. 메일 제공업체 비밀번호 변경됐는지 (앱 비밀번호 만료 가능)
2. 발송 한도 초과
3. 받는 이의 메일함 용량 초과
4. 아임웹 결제카드 만료 (요금제 끊김)

**Q. 회사 메일을 Google Workspace로 옮기고 싶은데 어떻게?**
A. 별도 프로세스. Google Workspace 가입 후 MX 레코드 변경 → 아임웹 SMTP 설정에 새 정보 입력.

**Q. 메일이 안 와도 SMS로 알림 받을 수 있나요?**
A. 가능합니다. 발신번호 등록 후 [알림톡·SMS 가이드](#) 참고.

**Q. 한 사이트에 여러 메일 주소를 쓸 수 있나요?**
A. 자동 메일 1개 주소만. 다만 **관리자 알림 메일**은 별도로 다른 주소 지정 가능 ("관리자가 다른 메일로 특정 알림만 받기" 기능).
